# AI辅助编程与工程化实训个人总结报告

## 一、项目概述

本次实训项目是一个名为「Todo Adventure」的待办事项管理应用。该应用融合了趣味决策机制，用户可以通过摇骰子或选数字的方式来决定是否执行待办事项。项目采用 Next.js + Flask 的前后端分离架构，使用 SQLite 作为数据库存储。

## 二、遇到的最大困难及解决过程

### 困难一：3D骰子动画效果实现

**问题描述**：在实现摇骰子功能时，需要创建一个逼真的3D骰子旋转动画效果。CSS 3D变换涉及 perspective、transform-style、rotateX/Y/Z 等多个属性的配合，参数设置不当会导致骰子变形或动画不流畅。

**解决过程**：
1. 首先查阅了CSS3 3D变换的相关资料，了解了立方体的构建原理
2. 设计了6个面的HTML结构，每个面对应不同的点数布局
3. 通过 transform 属性精确计算每个面的空间位置（translateZ + rotate）
4. 使用 perspective 属性设置观察者距离，增强3D透视感
5. 通过 CSS keyframes 和 transition 实现平滑的旋转动画
6. 调整动画时长和缓动函数，使骰子旋转更加自然

### 困难二：前后端联调与跨域问题

**问题描述**：前端调用后端API时遇到了跨域请求被阻止的问题，导致数据无法正常获取。

**解决过程**：
1. 分析错误信息，确认是跨域问题（CORS）
2. 在Flask后端安装并配置 flask-cors 扩展
3. 设置允许所有来源的跨域请求（开发环境）
4. 测试API接口，确认数据能正常传输
5. 考虑生产环境的安全性，准备在部署时配置具体的允许域名

### 困难三：项目部署方案

**问题描述**：考核要求提供可访问的线上Demo地址，但本地开发环境无法直接对外访问。

**解决过程**：
1. 研究了多种部署方案（Vercel、Render、Railway等）
2. 决定前端使用Vercel部署（Next.js官方推荐）
3. 后端使用Render部署（支持Flask应用）
4. 准备了部署配置文件和环境变量设置
5. 编写了详细的部署指南，方便后续操作

## 三、对AI辅助编程的心得体会

### AI辅助编程的优势

1. **快速原型开发**：AI能够根据需求描述快速生成完整的代码框架，大大缩短了项目初期的开发时间。在本项目中，AI帮助我快速搭建了Flask后端和Next.js前端的基础结构。

2. **学习辅助**：通过AI生成的代码示例，我学习到了很多新的技术和最佳实践。例如，3D骰子动画的实现方式、Tailwind CSS的高级用法、Next.js页面路由的配置等。

3. **解决问题效率高**：遇到技术问题时，AI能够快速提供解决方案和代码示例，减少了查阅资料的时间。

4. **代码质量保障**：AI生成的代码结构清晰，符合行业规范，减少了后期维护的难度。

### 需要注意的问题

1. **代码审查**：AI生成的代码虽然质量较高，但仍需要进行审查和优化。特别是涉及安全性和性能的部分，需要人工确认。

2. **理解业务需求**：AI只能根据提供的Prompt进行开发，需要开发者清晰地描述需求，避免产生误解。

3. **调试能力**：AI生成的代码可能存在一些细节问题，需要开发者具备一定的调试能力来解决。

4. **技术选型**：AI可能会推荐一些新技术或库，需要开发者根据项目实际情况进行评估和选择。
