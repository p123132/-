📊 Todo Adventure 数据库结构

users 表 - 用户信息表 4 条记录

字段名 类型 主键 说明
idINTEGER✓用户ID，自增主键
usernameTEXT-用户名
emailTEXT-邮箱地址
passwordTEXT-密码（bcrypt加密）
avatarTEXT-头像URL
roleTEXT-角色（admin/user）
created_atTIMESTAMP-创建时间
数据示例： admin (admin@example.com) - 管理员, testdemo, 11223, 112230 - 普通用户

todos 表 - 待办事项表 6 条记录

字段名 类型 主键 说明
idINTEGER✓待办ID，自增主键
user_idINTEGERFK关联用户ID
titleTEXT-待办标题
descriptionTEXT-待办描述
completedBOOLEAN-是否完成
priorityINTEGER-优先级（1-3）
categoryTEXT-分类（工作/学习/生活/其他）
due_dateDATE-截止日期
planned_dateDATE-计划日期
photoTEXT-照片URL
share_tokenTEXT-分享Token
created_atTIMESTAMP-创建时间
数据示例： 文科生高等教育（已完成）、早起、运动、阅读、喝水等待办事项

audit_logs 表 - 操作日志表 90 条记录

字段名 类型 主键 说明
idINTEGER✓日志ID，自增主键
user_idINTEGERFK操作用户ID
actionTEXT-操作类型
resourceTEXT-操作资源
resource_idINTEGER-资源ID
detailsTEXT-操作详情
ip_addressTEXT-IP地址
created_atTIMESTAMP-操作时间
记录的操作类型： user_register（用户注册）、user_login（用户登录）、todo_create（创建待办）、todo_update（更新待办）、todo_delete（删除待办）、todo_share（分享待办）等

数据库关系图

users  ──────────────── todos
  │                        │
  │ 1:N                    │ N:1
  │                        │
  ├─ id ──────→ user_id   │
  │                        │
audit_logs                │
  │                        │
  │ 1:N                    │
  ├─ user_id ←─── id      │
  │                        │
  └─ resource_id ←─── id ─┘

说明：
• users.id → todos.user_id：一个用户可以有多个待办事项
• users.id → audit_logs.user_id：一个用户可以有多个操作日志
• todos.id → audit_logs.resource_id：一个待办事项可以有多个相关日志
